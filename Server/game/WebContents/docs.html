/attack

GET ParameterDescription
id1The user ID of the attacker
id2The user ID of the target
sessionThe session ID of the current game
ReturnDescription
OLD HEALTH: <h1>
NEW HEALTH: <h2> 		Successful attack, the target went from <h1> to <h2> health
KILL: <loot>The attack killed id2, the attacker has gained <loot> resource points
The user changes their name to "DEAD", which will be read on next update by the app, handle death timeout, new name, etc.
Serverside, the user is reset to a basic zombie.
SESSION ENDED: <sID>The session has ended, a new session has started with session id <sID>
NONEXISTENT USEREither id1 or id2 do not exist.

/session

GET ParameterDescription
ReturnDescription
Session ID: <sID>
Session Length: <time>
Scoreboard: <...> 		Session info, lists users by amount of resources

/add

GET ParameterDescription
nameThe desired user name
xThe X coordinate of the user
yThe Y coordinate of the user
[type](optional) Defaults to human, "zombie" for zombie.
ReturnDescription
ID: <uID>
SESSION: <sID> 		Successful add, user with ID <uID> has been added to the game session <sID>

/list

GET ParameterDescription
[id]The userID of the user to display. If omitted, shows all
ReturnDescription
<uID>, <human>, <name>, <health>, <attack>, <defense>, <resources>, <locX>, <locY> Used to get own status, or for debugging

/update

GET ParameterDescription
idThe userID of the user to update
sessionThe game session of the user
[name]The new name of the user
[human]The new status of the user
[h]The new health of the user
[a]The new attack of the user
[d]The new defense of the user
[r]The new resources of the user
[x]The new x of the user
[y]The new y of the user
ReturnDescription
SELF|ALLY|ENEMY: <uID>, <human>, <name>, <health>, <attack>, <defense>, <resources>, <locX>, <locY>
ALLY|ENEMY: <...> 		Successful update, returns a list of nearby users.
SESSION ENDED: <sID>The session has ended, a new session has started with session id <sID>
NONEXISTENT USEREither id1 or id2 do not exist.
